드라곤 커브!

**접근 방식**

회전변환을 썼습니다.

Cos(-90) -sin(-90)
Sin(-90) cos(-90) 

 <hr/>
 
**어려웠던 점**

문제 이해를 제대로 못했던 것 같습니다. 문제 이해하는 데 시간이 오래 걸렸고
코딩 하는 데도 자꾸 문제를 제대로 정의하지 못해서 시간이 오래 걸렸습니다. 


<hr/>

**개념 정리**

이해만 하면 열심히 구현 하면 되는 것 같습니다.

<hr/>

**복잡도**
세그먼트트리 O(logN)

<hr/>

**피드백**

항상 느끼는 거지만 구현이 너무 어렵습니다. 앞으로는 문제의 풀이에 대해 생각하는 시간을 줄이고 일단 코딩부터 해봐야겠습니다. 
코딩을 하다보면 풀이만 생각할 때 보이지 않던 부분이 계속 보여서.. 코딩에 충분한 시간을 쓰지 못하는 것 같습니다.
1일차 문제 풀이 + 코딩 
2일차 문제 정답 확인 + 개념 정리
3일차 개념 적용 + 코딩 
으로 바꿔서 해보겠습니다.
