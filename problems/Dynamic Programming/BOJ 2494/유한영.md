**내가 이해한 문제내용**

숫자 맞추기
왼쪽으로 돌리기 : 해당 숫자 밑으로 다 돌아감
오른쪽으로 돌리기 : 해당 숫자만 돌아감.

 <hr/>
 
**접근 방식**

DP! 문제 보자마자 DP라고 생각했습니다. 
이건 Bottom-up으로 하면 될 것 같습니다.

dp[n][k] : n번째 숫자나사가 k번 왼쪽 회전한 상태 

 <hr/>
 
**어려웠던 점**

하지만 저걸 보고도 풀지는 못했습니다..
DP는 항상 점화식이 어렵습니다.

<hr/>

**개념 정리**

최소 회전수 뿐 아니라, 어디로 얼만큼 회전했는지를 출력해 줘야하므로 
매번 왼쪽 또는 오른쪽 중 어느방향으로 얼만큼 돌렸는지를 기록해두어야 한다.
DP에서 값 뿐만아니라 과정을 track하는 경우이다.

<hr/>

**복잡도**
O(MN)인 것 같네요.
