**접근 방식**

지난번 문제의 업그레이드라고 하길래 바로 세그먼트 트리를 공부했습니다.
전 문제는 세그먼트 트리를 사용하지 않았었기에, 세그먼트 트리를 사용해야할 거라고 생각해서 
세그먼트 트리 공부를 했습니다. 
https://www.acmicpc.net/blog/view/9
위의 블로그의 코드를 분석해서 활용했습니다. 
그리고 세그먼트 트리를 활용하여 범위에 따라 포함되는 범위가 가장 좁은 순서대로 나열하여 스위치를 켜주려고 했습니다.

 <hr/>
 
**어려웠던 점**
세그먼트 트리도 공부했고, 전에 사용한 pair과 unordered_map사용해서 정렬도 했는데, 
구현을 못하겠습니다.  


<hr/>

**개념 정리**
세그먼트 트리 - full binary tree
리프 노드는 전체 배열. 각 child 노드의 parent는 부분합을 가지고 있다. 
해당하는 범위를 찾으려면 

node가 담당하고 있는 구간이 [start,end] 
합을 구해야하는 구간이 [left,right]라면 
1. [left,right]  [start,end] 겹치지 않는 경우 -> 리턴 0
2. [left,right] 가 [start,end] 를 완전히 포함하는 경우 -> 리턴 노드
3. [start,end] 가 [left,right] 를 완전히 포함하는 경우 -> 다시 탐색!
4. [left,right] 와 [start,end] 가 겹쳐져 있는 경우 -> 다시 탐색!

<hr/>

**복잡도**
세그먼트트리 O(logN)
