내가 이해한 문제 내용
	-정수 n에 대하여 1,2,3의 합에 조합을 구하는 것(중복 가능)

접근 방식
	-숫자의 중복은 가능하며, 자리 바꾸는 것 까지 count해야해서, 모든 경우의 수를 생각해야했음
	반복문을 통해서 count하기 보다 모든 case를 따지는 recursive가 좋은 것 같다고 판단함

어려웠던 점
	-역시 recursive는 짜고나면 쉬워 보이지만, 짤때는 어렵다....정확하게 필요로하는 요만 넣고, return 값이 무엇인지 생각하자
	-문제에 대한 파악....처음 2개 이상으로만 나타내지는 것만 찾는 건줄알고, 1,2,3의 case는 무조건 0인줄 알았으나 알고보니 1개 이상이 었다....문제를 똑바로 읽자

복잡도
	-반복분은 입력값을 받기위해, 출력하기 사용했다...-->2n
	-일력값은 받기위한 1차 array하나만 선언함...-->n
